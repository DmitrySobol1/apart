Title: feature / Post detail — hourly chart + KPI — bucket=hour
Agent: Code Implementer

Goal
- Implement post detail view with hourly multi-series chart and KPI strip derived from the last metrics point in range.

Tech Requirements
- Technologies: React, SWR, Recharts, TypeScript, Tailwind
- Endpoint: `GET /channels/:username/posts/:messageId/metrics?from&to&bucket=hour`
- Controls: From/To date inputs (default last 30 days); persist in URL
- Series: views, reactionsTotal, forwards, replies
- KPI: take values from the last point in the returned series within [from, to]
- Empty/error/loading states with retry
- Size: ≤ 500 LOC
- Dependencies: stt-003, stt-004

Acceptance Criteria
- Chart renders hourly series correctly and responsively
- KPI header matches last data point within selected range
- Empty series shows a clear empty state; errors show a friendly message with Retry
- Back navigation returns to posts list preserving filters and scroll
