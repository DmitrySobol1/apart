## Goal

Implement the POST /api/booking endpoint with full request body validation using zod, console logging of booking payloads, and a stub success response (no real booking creation in MVP).

## Details

- Create `POST /api/booking` route in `backend/src/routes/booking.ts`.
- Request body schema (zod):
  - `dfrom`: required, string, format `DD-MM-YYYY`.
  - `dto`: required, string, format `DD-MM-YYYY`, must be after `dfrom`.
  - `planId`: required, positive integer.
  - `adults`: required, integer >= 1.
  - `roomTypeId`: required, non-empty string.
  - `guest.name`: required, non-empty string.
  - `guest.surname`: required, non-empty string.
  - `guest.phone`: required, format `+7XXXXXXXXXX` (12 chars, starts with `+7`, 10 digits after).
  - `guest.email`: required, valid email format.
  - `guest.notes`: optional string.
- On valid request:
  - Log full payload with timestamp to console.
  - Return 200 `{ success: true, message: "Request accepted" }`.
- On invalid request:
  - Return 400 with field-level error messages from zod.

## Tech Requirements

- Technologies: Express, TypeScript, zod.
- Files: `backend/src/routes/booking.ts`, `backend/src/types/index.ts` (BookingRequest type if needed).
- Dependencies: after stt-001 (project initialized).
- Constraints: ≤300 LOC.

## Acceptance Criteria

- Valid POST with all required fields returns 200 + `{ success: true, message: "Request accepted" }`.
- Full payload logged to server console with timestamp.
- Missing `guest.phone` returns 400 with descriptive error.
- Invalid phone format (not `+7XXXXXXXXXX`) returns 400.
- Invalid date format returns 400.
- `dto` before `dfrom` returns 400.
- No real booking created (MVP stub).
