## Goal

Build the BookingPage (Step 3) with guest data form, booking summary sidebar, and validation; and the ConfirmationPage (Step 4) with success message and state reset.

## Details

- **GuestForm component** (`components/GuestForm.tsx`):
  - Fields: first name (required), last name (required), phone (required, masked `+7(___) ___-__-__`), email (required, email format), notes (optional, textarea).
  - Client-side validation on blur + on submit.
  - Error messages shown below each invalid field.
  - Phone mask forces `+7` prefix; before submission, strip formatting to clean `+7XXXXXXXXXX`.
- **BookingSummary component** (`components/BookingSummary.tsx`):
  - Displays: room photo (first from gallery), room name, check-in/check-out dates, number of guests, rate plan name, total price.
  - Read-only summary sidebar.
- **BookingPage** (`pages/BookingPage.tsx`):
  - Two-column layout: left — GuestForm; right — BookingSummary.
  - Agreement checkbox: "I agree to the terms" (required before submit).
  - "Book" button: disabled until form valid + checkbox checked.
  - "Back to Rooms" link.
  - On submit: POST to `/api/booking` with BookingRequest body.
  - Loading: spinner on "Book" button; form disabled during submission.
  - Error: inline message if POST fails ("Booking request failed. Please try again.").
  - On success: store guest data in context, navigate to `/confirmation`.
- **ConfirmationPage** (`pages/ConfirmationPage.tsx`):
  - Centered success card.
  - Checkmark icon + "Request Received!" heading.
  - "Thank you for your request. We will contact you shortly."
  - Booking summary: room name, dates, guest name.
  - "Back to Search" button: calls `reset()` on BookingContext, navigates to `/`.
  - No loading/error states (static page).

## Tech Requirements

- Technologies: React, TypeScript, Tailwind CSS.
- Libraries: phone mask library (e.g., `react-input-mask` or custom implementation).
- Files: `frontend/src/pages/BookingPage.tsx`, `frontend/src/pages/ConfirmationPage.tsx`, `frontend/src/components/GuestForm.tsx`, `frontend/src/components/BookingSummary.tsx`.
- Dependencies: after stt-004 (types, context, API client); after stt-003 (POST endpoint ready).
- Constraints: ≤600 LOC total across new files.

## Acceptance Criteria

- Form validates all required fields on blur and on submit.
- Phone mask shows `+7(___) ___-__-__`; submitted value is clean `+7XXXXXXXXXX`.
- Invalid fields show error messages below inputs.
- "Book" button disabled until form valid + checkbox checked.
- Valid submission shows loading spinner, then navigates to confirmation page.
- POST failure shows inline error message.
- Confirmation page displays booking summary.
- "Back to Search" resets all context state and navigates to `/`.
- "Back to Rooms" link works on booking page.
