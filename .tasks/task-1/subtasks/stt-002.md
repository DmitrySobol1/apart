## Goal

Implement the backend Bnovo proxy layer: HTTP client for upstream API, 4 GET proxy routes, input validation, in-memory caching, error handling, and request timeouts.

## Details

- Implement `bnovoClient.ts` service:
  - Reads `BNOVO_UID`, `BNOVO_ACCOUNT_ID`, `BNOVO_API_BASE` from environment.
  - Axios instance with 10s timeout for all Bnovo requests.
  - Methods: `getRooms(dfrom, dto)`, `getPlans()`, `getAmenities()`, `getAccount()`.
- Create 4 proxy routes:
  - `GET /api/rooms?dfrom=DD-MM-YYYY&dto=DD-MM-YYYY` → Bnovo `/rooms?account_id={env}&dfrom=...&dto=...`
  - `GET /api/plans` → Bnovo `/plans?account_id={env}`
  - `GET /api/amenities` → Bnovo `/amenities`
  - `GET /api/account` → Bnovo `/accounts?uid={env}`
- Input validation for `/api/rooms`:
  - `dfrom` and `dto` required, format `DD-MM-YYYY`.
  - `dto` must be after `dfrom`.
  - Return 400 with descriptive error for invalid input.
- In-memory cache for `/rooms` responses:
  - Key: `dfrom+dto` string.
  - TTL: 5 minutes.
  - Simple Map-based implementation.
- Error handler middleware:
  - Bnovo errors → 502 `{ error: "Upstream API error" }`.
  - Bnovo timeout → 504 `{ error: "Upstream API timeout" }`.
  - Validation errors → 400 `{ error: "..." }`.
  - Log all errors with timestamp.

## Tech Requirements

- Technologies: Express, TypeScript, Axios.
- Files: `backend/src/services/bnovoClient.ts`, `backend/src/routes/rooms.ts`, `backend/src/routes/plans.ts`, `backend/src/routes/amenities.ts`, `backend/src/routes/account.ts`, `backend/src/middleware/errorHandler.ts`.
- Dependencies: after stt-001 (project initialized); after stt-000 (API structure known).
- Constraints: ≤600 LOC per file.

## Acceptance Criteria

- `curl http://localhost:3000/api/rooms?dfrom=01-03-2026&dto=03-03-2026` returns Bnovo room data.
- `curl http://localhost:3000/api/plans` returns rate plans.
- `curl http://localhost:3000/api/amenities` returns amenity definitions.
- `curl http://localhost:3000/api/account` returns hotel info.
- Invalid dates return 400 with error message.
- Second identical `/api/rooms` request served from cache (faster response, no upstream call).
- Bnovo timeout returns 504; Bnovo error returns 502.

## Цель

Реализовать backend-прокси для Bnovo: HTTP-клиент, 4 GET-маршрута, валидацию, кэширование, обработку ошибок и таймауты.

## Критерии приёмки
