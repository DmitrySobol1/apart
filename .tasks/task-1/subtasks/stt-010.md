# stt-010 | Code Implementer | infra / Tooling & config alignment — linters + project-commands + agent configs + hooks

## Goal

After project initialization (stt-001), configure development tooling and update all configuration files inherited from the template project (TG Channel Stats) to match the apart-nn stack and project structure.

## Context

The repository was created from a template. Agent configs, hooks, steerings, and project-commands currently reference commands and paths from a different project. This subtask brings everything into alignment with the actual apart-nn project.

## Tech Requirements

- ESLint + Prettier configured for both `frontend/` and `backend/`
- Quality check script (`qc` or equivalent) in package.json
- All `.claude/` and `.memory-bank/` configs referencing correct commands

## Acceptance Criteria

### Linters & Quality Scripts
- [ ] ESLint configured for both `frontend/` and `backend/` with TypeScript support
- [ ] Prettier configured for both packages
- [ ] `npm run lint` / `npm run format` scripts work in both packages
- [ ] A combined quality check script exists (e.g., `npm run qc` = lint + type-check)
- [ ] `npm run build` works in both packages

### `.memory-bank/steerings/project-commands.md`
- [ ] All placeholder `# TODO` commands replaced with actual working commands
- [ ] Command reference table updated with correct commands

### `.claude/agents/` configs
- [ ] `code-implementer.md` — path `apps/specs/index.md` removed or replaced with correct path
- [ ] All agents referencing `project-commands.md` will now find correct commands (no code changes needed in agents if project-commands.md is updated)

### `.claude/hooks.json`
- [ ] `pnpm qc:wrapped` replaced with actual quality check command
- [ ] `pnpm code-style` replaced with actual formatting command (e.g., `npx prettier --write`)
- [ ] All hooks execute successfully without errors

## Refs
