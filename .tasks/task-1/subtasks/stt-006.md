## Goal

Build the RoomsPage (Step 2): room catalog displaying filtered and sorted room cards with photo galleries, amenity lists, rate plan selectors, and booking action.

## Details

- **PhotoGallery component** (`components/PhotoGallery.tsx`):
  - Displays room photos sorted by `order` field.
  - Prev/next arrow navigation.
  - First photo visible by default.
- **AmenityList component** (`components/AmenityList.tsx`):
  - Row of amenity icons with tooltip names.
  - Handles missing icons gracefully.
- **PlanSelector component** (`components/PlanSelector.tsx`):
  - Dropdown if room has multiple rate plans.
  - Default: first plan selected.
  - Selecting a plan updates displayed price.
- **RoomCard component** (`components/RoomCard.tsx`):
  - Layout: left — PhotoGallery; right — room name, area (m²), max guests, AmenityList, PlanSelector, total price for period, available count, "Book" button.
  - "Book" button stores selected room + plan in BookingContext, navigates to `/booking`.
- **RoomsPage** (`pages/RoomsPage.tsx`):
  - Header with search summary: "Check-in: ..., Check-out: ..., Guests: ...".
  - "Back to Search" link.
  - Vertical list of RoomCard components.
  - Client-side filtering: only rooms with `available > 0` **and** `maxGuests >= searchParams.adults`.
  - Sort by price ascending (lowest plan price).
  - Empty state: "No rooms available for selected dates. Try different dates." with "Back to Search" link.

## Tech Requirements

- Technologies: React, TypeScript, Tailwind CSS.
- Files: `frontend/src/pages/RoomsPage.tsx`, `frontend/src/components/RoomCard.tsx`, `frontend/src/components/PhotoGallery.tsx`, `frontend/src/components/AmenityList.tsx`, `frontend/src/components/PlanSelector.tsx`.
- Dependencies: after stt-004 (types, context ready); after stt-005 (SearchPage stores data).
- Constraints: ≤600 LOC total across new files.

## Acceptance Criteria

- Rooms page shows filtered cards (only rooms fitting guest count and available > 0).
- Cards sorted by price ascending.
- Photo gallery navigates between photos with arrows.
- Plan selector changes displayed price.
- "Book" stores selection in context and navigates to `/booking`.
- Empty state shown when no rooms match criteria.
- "Back to Search" link works.
- Search summary header displays correct dates and guest count.
