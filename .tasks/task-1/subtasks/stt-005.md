## Goal

Build the SearchPage (Step 1): date selection for check-in/check-out, guest count selector, search action calling the backend API, loading/error states, and navigation to rooms page.

## Details

- **DatePicker component** (`components/DatePicker.tsx`):
  - Check-in date picker (required, default: today).
  - Check-out date picker (required, default: today + 1).
  - Check-out must be > check-in; auto-adjust if user sets check-out <= check-in.
  - Use `react-datepicker` library or native date inputs.
- **GuestCounter component** (`components/GuestCounter.tsx`):
  - +/- buttons, integer value.
  - Min: 1, Max: 10, Default: 2.
- **SearchPage** (`pages/SearchPage.tsx`):
  - Centered form card layout.
  - Header: hotel name (fetched from `GET /api/account` on mount).
  - Form fields vertically stacked: check-in, check-out, guests, "Search" button.
  - On "Search": call `GET /api/rooms?dfrom=DD-MM-YYYY&dto=DD-MM-YYYY`.
  - On success: store search params + rooms in BookingContext, navigate to `/rooms`.
  - Loading state: spinner on button, form disabled.
  - Error state: inline message below form ("Failed to load rooms. Please try again.") with retry action.
  - If 0 rooms returned: navigate to `/rooms` (shows empty state there).

## Tech Requirements

- Technologies: React, TypeScript, Tailwind CSS.
- Libraries: `react-datepicker` (or native inputs), Axios client from stt-004.
- Files: `frontend/src/pages/SearchPage.tsx`, `frontend/src/components/DatePicker.tsx`, `frontend/src/components/GuestCounter.tsx`.
- Dependencies: after stt-004 (types, context, API client, routing ready).
- Constraints: ≤400 LOC total across new/modified files.

## Acceptance Criteria

- User selects dates and guest count, clicks "Search".
- Loading spinner shows on button; form is disabled during request.
- On success: navigates to `/rooms` with data in BookingContext.
- On API error: inline error message with retry option.
- Check-out auto-adjusts if set <= check-in.
- Guest counter respects min/max bounds.
- Hotel name displayed in header (from /api/account).
