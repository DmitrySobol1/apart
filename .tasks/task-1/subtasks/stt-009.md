## Goal

Verify the complete booking widget works end-to-end: full user flow through all 4 steps, iframe embedding, edge cases, backend caching, and credential isolation.

## Details

- **Full flow verification** (with real Bnovo API via backend proxy):
  1. Search: select dates + guests → click "Search" → rooms load.
  2. Room catalog: cards display with photos, amenities, plans, prices → select room + plan → click "Book".
  3. Guest form: fill all fields (name, surname, +7 phone, email) → check agreement → click "Book" → POST succeeds.
  4. Confirmation: "Request Received!" shown with booking summary → "Back to Search" resets state.
- **Edge case testing**:
  - No available rooms for selected dates → empty state message shown.
  - Invalid date selection (check-out <= check-in) → auto-correction or error.
  - Empty required guest fields → validation errors shown.
  - Direct URL access to `/rooms` or `/booking` without prior search → redirect to `/`.
  - Browser back button navigation within the flow.
- **iframe testing**:
  - Load widget in `test-iframe.html`.
  - Verify auto-height adjusts on navigation and content changes.
  - Verify no double scrollbars.
- **Backend caching**:
  - Make two identical `/api/rooms` requests → verify second is faster (served from cache).
- **Credential isolation**:
  - Inspect frontend bundle (dist) → no `BNOVO_UID` or `BNOVO_ACCOUNT_ID` strings.
  - Inspect browser network requests → no Bnovo identifiers visible in frontend requests.

## Tech Requirements

- Tools: browser, curl, developer tools (Network tab, search in sources).
- Manual testing; no automated test framework required.
- Dependencies: after stt-008 (all features built and styled).

## Acceptance Criteria

- Complete flow works end-to-end (search → rooms → booking → confirmation).
- All edge cases handled gracefully (no crashes, proper error messages).
- Widget works correctly inside iframe with auto-height.
- Backend cache improves response time on repeated requests.
- No Bnovo credentials (UID, account_id) present in frontend bundle or client-visible network requests.
- Booking payload logged correctly on server console.
