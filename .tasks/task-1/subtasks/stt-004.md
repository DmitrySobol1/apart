## Goal

Build the frontend foundation: TypeScript type definitions, Axios API client with error interceptor, BookingContext for cross-page state, React Router setup with 4 routes and stub pages, and redirect guards.

## Details

- Create TypeScript types in `frontend/src/types/index.ts`:
  - `Room`, `RoomPhoto`, `Amenity`, `RoomPlan`, `GuestData`, `BookingRequest`, `BookingResponse` (per plan Types section).
  - Room `id` is `string` type.
- Implement Axios client in `frontend/src/api/client.ts`:
  - Base URL from `VITE_API_BASE_URL` (or relative `/api` in production).
  - JSON content type defaults.
  - Response interceptor mapping errors to user-friendly messages:
    - Network error → "Connection error. Check your internet and try again."
    - 400 → "Invalid request. Please check your input."
    - 502/504 → "Service temporarily unavailable. Please try again in a moment."
    - 5xx → "Something went wrong. Please try again."
- Create `BookingContext` in `frontend/src/context/BookingContext.tsx`:
  - State: `searchParams`, `rooms`, `selectedRoom`, `selectedPlan`, `guest` (all nullable).
  - Actions: `setSearchParams`, `setRooms`, `selectRoom`, `selectPlan`, `setGuest`, `reset`.
- Set up React Router in `App.tsx`:
  - Routes: `/` → SearchPage, `/rooms` → RoomsPage, `/booking` → BookingPage, `/confirmation` → ConfirmationPage.
  - Stub pages with placeholder text.
  - Redirect guard: `/rooms` and `/booking` redirect to `/` if `searchParams` is null in context.

## Tech Requirements

- Technologies: React 18, TypeScript, React Router v6, Axios.
- Files: `frontend/src/types/index.ts`, `frontend/src/api/client.ts`, `frontend/src/context/BookingContext.tsx`, `frontend/src/App.tsx`, `frontend/src/pages/SearchPage.tsx`, `frontend/src/pages/RoomsPage.tsx`, `frontend/src/pages/BookingPage.tsx`, `frontend/src/pages/ConfirmationPage.tsx`.
- Dependencies: after stt-001 (project initialized); may be updated after stt-000 if API data differs.
- Constraints: ≤400 LOC total across new files.

## Acceptance Criteria

- App renders SearchPage at `/`.
- Navigating to `/rooms` without search data redirects to `/`.
- Navigating to `/booking` without search data redirects to `/`.
- BookingContext provides all state fields and actions.
- Axios client configured with error interceptor and importable from any component.
- All TypeScript types defined and exportable.
- No TypeScript compilation errors.
