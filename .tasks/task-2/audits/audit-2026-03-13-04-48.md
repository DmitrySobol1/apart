## Previous Issues Resolution Check

### MAJOR Issue 1 — `bnovoId` type in `admin/src/types/index.ts`

**Previous finding:** `bnovoId: number` in both `Room` and `Coefficient` interfaces. Backend stores and returns `bnovoId` as `string`. Type contract was wrong.

**Verification:**

```typescript
// admin/src/types/index.ts — current state
export interface Room {
  bnovoId: string;   // FIXED: was number
  name: string;
}

export interface Coefficient {
  bnovoId: string;   // FIXED: was number
  roomName: string;
  coefficient1: number;
  coefficient2: number;
  coefficient3: number;
  updatedAt: string;
}
```

**Status: RESOLVED.** Both `Room.bnovoId` and `Coefficient.bnovoId` are now correctly typed as `string`, matching the Mongoose schema (`type: String`) and the Bnovo API's string room IDs.

Cascade effect on `CoefficientsPage.tsx`: `TableState = Record<string, RowState>` (was `Record<number, RowState>`), `updateCell(bnovoId: string, ...)`, `handleBlur(bnovoId: string, ...)`, `patchCoefficient(bnovoId: string, ...)` — all confirmed consistent with the fixed type.

### MAJOR Issue 2 — API response envelope unwrapping in `admin/src/api/client.ts`

**Previous finding:** Backend returns `{ data: [...] }` envelope. Client was doing `api.get<T>(...).then(r => r.data)`, resolving to the envelope object instead of the array. `CoefficientsPage.sort()` call would crash at runtime.

**Verification:**

```typescript
// admin/src/api/client.ts — current state
export const getRooms = (): Promise<Room[]> =>
  api.get<{ data: Room[] }>('/rooms').then((r) => r.data.data);
  // FIXED: generic changed to { data: Room[] }, unwrap via r.data.data

export const getCoefficients = (): Promise<Coefficient[]> =>
  api.get<{ data: Coefficient[] }>('/coefficients').then((r) => r.data.data);
  // FIXED: generic changed to { data: Coefficient[] }, unwrap via r.data.data

export const patchCoefficient = (
  bnovoId: string,   // FIXED: was number
  data: Partial<Pick<Coefficient, 'coefficient1' | 'coefficient2' | 'coefficient3'>>,
): Promise<Coefficient> =>
  api.patch<{ data: Coefficient }>(`/coefficients/${bnovoId}`, data).then((r) => r.data.data);
  // FIXED: generic and unwrap corrected, bnovoId parameter type fixed to string
```

**Status: RESOLVED.** All three API client functions now correctly unwrap the `{ data: [...] }` envelope. The generics accurately reflect the actual response shape. The `CoefficientsPage` will receive a real `Coefficient[]` array and the `.sort()` call will work correctly at runtime.

### MINOR Issue — Deprecated `{ new: false }` Mongoose option in `room-sync.ts`

**Previous finding:** `findOneAndUpdate` called with `{ new: false }`, which generates a Mongoose deprecation warning. Should use `{ returnDocument: 'before' }`.

**Verification:**

```typescript
// backend/src/services/room-sync.ts — current state (line 65-69)
const result = await Room.findOneAndUpdate(
  { bnovoId },
  { name },
  { upsert: true, returnDocument: 'before', lean: true },
);
```

**Status: RESOLVED.** The deprecated `{ new: false }` option has been replaced with `{ returnDocument: 'before' }`. No Mongoose deprecation warning will be generated.
