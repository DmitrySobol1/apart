## feature / Config + minimal logging/errors — Zod-validated config — concise logs

### Goal
Create a Zod-validated config module and add minimal progress logging and clear MTProto error mapping.

### Tech Requirements
- Libraries: `zod`, Node env, Drizzle ORM.
- Depends on: stt-001 (zod baseline), used by all route modules.
- Size constraints: ≤3 files, ≤200 LOC.

### Acceptance Criteria
- Config module validates env: `API_ID`, `API_HASH`, `SESSION_*`, `DB_PATH`, `BASIC_AUTH_*` and exports a typed config.
- Minimal logs: start/end, per-page progress, added/updated counts; no noisy debug.
- Map common errors to safe messages (e.g., `FLOOD_WAIT_X`, `ACCESS_HASH_INVALID`).
- Verification commands:
  - `pnpm --filter @tg-channel-stats/backend type-check`
  - `pnpm --filter @tg-channel-stats/backend build`
- Artifacts:
  - `packages/backend/src/config.ts`
  - Optional: small logging util `packages/backend/src/log.ts`
