## feature / Sync ingestion orchestrator + admin route — POST /admin/ingest/:username — Basic Auth

### Goal
Implement sync ingestion (newest→oldest with rescan) and expose it via an admin-only route returning added/updated counts.

### Tech Requirements
- Libraries: Hono, Drizzle ORM, gram.js, `zod`/`drizzle-zod`.
- Depends on: stt-001 (auth + schemas), stt-002 (resolver), stt-003 (post upsert/snapshot).
- Size constraints: ≤6 files, ≤400 LOC.

### Acceptance Criteria
- Orchestrator respects `maxMessages` and `rescanLatest`, updates ingestion summary.
- Admin route `POST /admin/ingest/:username` validates params/body and triggers orchestrator.
- Response shape `{ ok, channel, added, updated }` validated via Zod.
- Verification commands:
  - `pnpm --filter @tg-channel-stats/backend type-check`
  - `pnpm --filter @tg-channel-stats/backend build`
- Artifacts:
  - `packages/backend/src/ingest/orchestrate.ts`
  - `packages/backend/src/routes/admin.ts`
