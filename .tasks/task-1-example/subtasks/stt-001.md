## feature / Foundations — validation + Basic Auth — drizzle-zod

### Goal
Establish validation and access control foundations: derive Zod schemas from Drizzle and secure admin endpoints with Basic Auth.

### Tech Requirements
- Libraries: `zod`, `drizzle-zod`, Hono middleware for Basic Auth (custom minimal).
- Depends on: `.tasks/task-1/plan.md`.
- Size constraints: ≤5 files, ≤300 LOC.

### Acceptance Criteria
- Zod + drizzle-zod installed in backend; base schemas exported:
  - `ChannelRowSchema`, `PostRowSchema`, `PostMetricsHourlyRowSchema` (derived via `createSelectSchema`).
  - DTOs via `.pick()/.omit()`: `PostListItemSchema`, `PostMetricsPointSchema` placeholders.
- Basic Auth middleware present; protects all `/admin/*` routes.
- Verification commands:
  - `pnpm --filter @tg-channel-stats/backend type-check`
  - `pnpm --filter @tg-channel-stats/backend build`
- Artifacts:
  - Backend schema/validation module created (e.g., `packages/backend/src/schemas/*.ts`).
  - Basic Auth middleware file (e.g., `packages/backend/src/middleware/basic-auth.ts`).
