## feature / MTProto client + channel resolver — createTelegramClient & resolveChannel — sync

### Goal
Create an MTProto Telegram client and a resolver that maps `@username` to a channel peer, upserting the channel into the database.

### Tech Requirements
- Libraries: `telegram` (gram.js), Node env access.
- Depends on: stt-001 (validation types available), `.env` values (`API_ID`, `API_HASH`, `SESSION_*`).
- Size constraints: ≤4 files, ≤300 LOC.

### Acceptance Criteria
- Helper `createTelegramClient()` initializes client from env and connects.
- Function `resolveChannel(username)` returns `{ tgId, accessHash, title }` and internal `channels.id` after upsert.
- Handles `ACCESS_HASH_INVALID` by re-resolving once.
- Verification commands:
  - `pnpm --filter @tg-channel-stats/backend type-check`
- Artifacts:
  - `packages/backend/src/mtproto/client.ts`
  - `packages/backend/src/mtproto/resolve.ts`
