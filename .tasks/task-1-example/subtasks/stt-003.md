## feature / Post mapping + upsert + snapshot — posts + post_metrics_hourly — minimal

### Goal
Map MTProto messages into `posts` rows, upsert latest fields, and write an hourly snapshot in `post_metrics_hourly`.

### Tech Requirements
- Libraries: `telegram` (gram.js), Drizzle ORM, `zod`/`drizzle-zod` DTOs.
- Depends on: stt-001 (schemas), stt-002 (client + resolver).
- Size constraints: ≤6 files, ≤400 LOC.

### Acceptance Criteria
- Mapper converts MTProto message to post payload (content, flags, counters, reactions JSON breakdown).
- Upsert by `(channel_id, message_id)` updates content and latest counters.
- Snapshot write computes `ts_hour` and upserts `(channel_id, message_id, ts_hour)` with counters.
- Verification commands:
  - `pnpm --filter @tg-channel-stats/backend type-check`
- Artifacts:
  - `packages/backend/src/ingest/mapMessage.ts`
  - `packages/backend/src/ingest/upsertPost.ts`
  - `packages/backend/src/ingest/writeSnapshot.ts`
