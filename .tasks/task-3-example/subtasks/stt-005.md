## Goal

Validate API input for `since`: ensure ISO 8601 format, reject future dates with HTTP 400, and confirm error mapping is user-friendly.

## Details

- Add tests that POST to `/api/admin/ingest?username=...` with invalid `since` (bad format) and future `since`.
- Assert HTTP 400 and presence of a clear `message` field in response JSON.
- Verify OpenAPI `/api/docs` (or `/api/openapi.json`) includes `since` in the schema with an example.

## Tech Requirements

- Libraries: Vitest (backend), supertest or fetch-based test harness.
- Files: tests under `packages/backend/` (e.g., `src/routes/__tests__/admin.since.spec.ts`).
- Dependencies: after stt-002 (contract implemented).
- Constraints: ≤600 LOC per file; do not add new runtime dependencies unless necessary.

## Acceptance Criteria

- Tests cover invalid format and future date cases; both return 400 with friendly JSON errors.
- OpenAPI contains `since` with an example value.
- CI/type-check passes for backend package.
