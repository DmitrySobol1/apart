## Goal

Unit test the orchestrator’s `since` boundary logic: early-stop when encountering older pages, update only posts within the boundary, and honor an optional `maxMessages` cap.

## Details

- Mock gram.js `TelegramClient.invoke(GetHistory)` to return descending pages with mixed dates around the boundary.
- Assert that only posts with `date >= since` are upserted/snapshotted.
- Verify paging stops once a page yields no qualifying messages.
- If `maxMessages` is provided, ensure total processed does not exceed it.

## Tech Requirements

- Libraries: Vitest, test doubles for Telegram client; in-memory or mocked DB via Drizzle (or stubbed `DbClient`).
- Files: tests under `packages/backend/src/ingest/__tests__/orchestrate.since.spec.ts`.
- Dependencies: after stt-003 (orchestrator updated).
- Constraints: ≤600 LOC per file; no network calls in tests.

## Acceptance Criteria

- Tests pass demonstrating: (1) early-stop at boundary, (2) only `date >= since` processed, (3) `maxMessages` acts as cap.
- No regressions to rate-limit/backoff behavior (simulate a FLOOD_WAIT error path once).
