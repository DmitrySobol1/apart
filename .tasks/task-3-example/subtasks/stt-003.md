## Goal

Implement `since` boundary handling in the ingestion orchestrator: process posts with `date >= since`, early-stop when older pages are reached, and clamp `rescanLatest` to the [since, now] window.

## Details

- Add `since?: Date` to orchestrator options and pass from route handler.
- During paging, only upsert/snapshot messages with `mapped.date >= since`.
- Page stop rule: if a page yields no qualifying messages (all `< since`), stop further paging.
- Clamp `rescanLatest` so that posts older than `since` are not refreshed.
- Keep current `maxId`-based paging; do not implement `offsetDate` yet.

## Tech Requirements

- Libraries: gram.js (Telegram Core API), Drizzle ORM.
- Files: `packages/backend/src/ingest/orchestrate.ts` (primary), `packages/backend/src/routes/admin.ts` (wire option).
- Dependencies: after stt-002 (API contract in place).
- Constraints: ≤600 LOC per file; preserve existing logging and error mapping.

## Acceptance Criteria

- Given a `since` date, orchestrator only adds/updates posts with `date >= since`.
- Paging stops once older-than-since pages are encountered.
- `rescanLatest` does not update posts older than `since`.
- No regressions to rate-limit/backoff logic and access-hash refresh.
