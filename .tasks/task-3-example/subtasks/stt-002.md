## Goal

Extend the admin ingest API contract to accept `since` as an ISO 8601 datetime, validate it, and expose it in OpenAPI.

## Details

- Update `IngestBodySchema` to include `since?: z.string().datetime()` with examples.
- Reject future `since` values with HTTP 400 in the route; normalize to UTC internally.
- Document precedence: `since` is a lower-bound filter; `maxMessages` remains an optional safety cap.
- OpenAPI should display `since` with an example in `/api/openapi.json` and `/api/docs`.

## Tech Requirements

- Libraries: `zod`, `@hono/zod-openapi`.
- Files: `packages/backend/src/schemas/index.ts`, `packages/backend/src/routes/admin.ts`.
- Dependencies: after stt-001 (baseline); before stt-003 and stt-004.
- Constraints: ≤600 LOC per file; reuse derived schemas; consistent error mapping.

## Acceptance Criteria

- OpenAPI shows `since` param in admin ingest request body with ISO example.
- Invalid/future `since` returns HTTP 400 with a friendly message and machine-parsable JSON shape.
- No regressions to existing fields (`maxMessages`, `rescanLatest`).
