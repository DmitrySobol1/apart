## Goal

Establish a baseline of current admin ingest behavior (without `since`) by recording API responses and observed DB effects to compare after changes.

## Details

- Use the existing `POST /api/admin/ingest?username=<name>` without `since` to ingest a known test channel.
- Capture response payload (`added`, `updated`, `channel`) and approximate elapsed time.
- Note current defaults for `rescanLatest` and any implicit caps.
- Optional: observe WAL growth in `data/stats.sqlite-wal` during ingest to confirm writes.

## Tech Requirements

- Tools: curl or HTTP client; shell access to view `data/` files.
- No code changes; manual evaluation only.
- Dependencies: none (first in sequence).
- Constraints: ≤600 LOC touched per file (no code expected).

## Acceptance Criteria

- One successful ingest run recorded with raw request and response.
- A brief note of observed behavior: added/updated counts and whether recent posts were rescanned.
- Collected notes stored alongside task artifacts (e.g., commit message or project log).
