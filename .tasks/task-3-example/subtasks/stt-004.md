## Goal

Add a “By date” mode in the web admin UI that triggers ingest with a hardcoded `since = 2025-06-01T00:00:00Z` and keeps `rescanLatest` configurable.

## Details

- Introduce UI toggle: “By count” vs “By date”.
- In “By date” mode, do not render a date input; always send `since = "2025-06-01T00:00:00Z"`.
- Hide `maxMessages` control by default; optionally expose as an advanced safety cap.
- Keep existing success/error toasts, spinner, and post-sync refresh.

## Tech Requirements

- Libraries: React, TypeScript, shadcn/ui, SWR.
- Files: `packages/web/src/lib/api.ts` (extend params), `packages/web/src/pages/HomePage.tsx` (controls and action).
- Dependencies: after stt-002 (API contract ready).
- Constraints: ≤600 LOC per file; maintain theme/dark mode consistency.

## Acceptance Criteria

- Toggle exists; selecting “By date” triggers a POST with `since = 2025-06-01T00:00:00Z`.
- `rescanLatest` can be set; `maxMessages` hidden by default in “By date”.
- UI shows proper loading and success/error toasts; channel list refreshes on success.
