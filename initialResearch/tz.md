# Техническое задание на разработку виджета онлайн-бронирования

**Приложение к договору на разработку**

---

## 1. Общие сведения

### 1.1 Наименование проекта

Виджет онлайн-бронирования апарт-отеля.

### 1.2 Назначение

Разработка виджета для онлайн-бронирования номеров апарт-отеля, размещаемого на сайте Заказчика. Виджет обеспечивает полный цикл бронирования: от выбора дат и номера до оформления брони с последующей онлайн-оплатой.

### 1.3 Источник данных

Виджет получает данные о номерах, ценах и доступности через публичный API сервиса Bnovo (платформа управления отелем, используемая Заказчиком). Создание бронирований осуществляется через модуль бронирования Bnovo.

---

## 2. Состав разработки

Разработка включает две части:

| Часть | Описание |
|-------|----------|
| **Frontend** | Клиентская часть виджета (пользовательский интерфейс), отображаемая на сайте Заказчика |
| **Backend** | Серверная часть с административной панелью управления виджетом |

---

## 3. Frontend — виджет бронирования

### 3.1 Технология

Виджет реализуется как одностраничное React-приложение, встраиваемое на сайт Заказчика через iframe. Виджет совместим с любым сайтом независимо от его технологии (HTML, WordPress, Tilda, иные CMS и конструкторы).

### 3.2 Код встраивания

Заказчику предоставляется фрагмент кода (iframe или JavaScript-загрузчик) для размещения виджета на сайте.

### 3.3 Функциональные требования к виджету

#### 3.3.1 Выбор дат и параметров

- Календарь выбора даты заезда и выезда.
- Отображение минимальных цен за ночь в ячейках календаря.
- Отображение закрытых (недоступных) дат.
- Выбор количества гостей (взрослые).
- Поле для ввода промокода.

#### 3.3.2 Каталог номеров

- Отображение списка доступных номеров на выбранные даты.
- Карточка номера: фотографии (галерея с возможностью листания), название, описание, площадь, список удобств (с иконками), вместимость.
- Отображение цены за выбранный период.
- Отображение тарифных планов для каждого номера (например, «Бронирование онлайн», «Невозвратный со скидкой 10%») с указанием условий отмены и стоимости.
- Отображение количества свободных номеров данного типа.

#### 3.3.3 Оформление бронирования

- Форма ввода данных гостя: имя, фамилия, телефон, email, примечания.
- Отображение итоговой стоимости бронирования.
- Кнопка подтверждения бронирования.
- После оформления — перенаправление на страницу онлайн-оплаты (платёжный шлюз, подключённый в Bnovo).

#### 3.3.4 Дополнительные услуги

- Отображение списка дополнительных услуг отеля (питание, парковка, оборудование и пр.).
- Возможность добавить услуги к бронированию с указанием количества.

#### 3.3.5 Общие требования к интерфейсу

- Адаптивная вёрстка (корректное отображение на десктопных и мобильных устройствах).
- Автоподстройка высоты iframe под содержимое.

## 4. Backend — серверная часть и административная панель

### 4.1 Технология

Серверная часть реализуется на Node.js.

### 4.2 Серверная логика (API-прокси)

Backend выполняет роль посредника между виджетом и сервисом Bnovo:

- Проксирование запросов на получение данных (номера, цены, доступность, услуги) из публичного API Bnovo.
- Отправка запроса на создание бронирования в модуль бронирования Bnovo.
- Получение номера брони и ссылки на оплату из ответа Bnovo, передача их обратно в виджет.

### 4.3 Административная панель

Веб-интерфейс для управления виджетом со следующим функционалом:

#### 4.3.1 Настройки виджета

- Настройка внешнего вида: цветовая схема, логотип.
- Настройка текстов и заголовков виджета.
- Включение/отключение отдельных блоков (промокод, дополнительные услуги, примечания).

#### 4.3.2 Управление данными подключения

- Настройка идентификаторов объекта Bnovo (UID, account_id).
- Выбор тарифных планов, отображаемых в виджете.
- Настройка типа гарантии бронирования (онлайн-оплата / без предоплаты).

#### 4.3.3 Мониторинг

- Журнал созданных через виджет бронирований (номер брони, дата создания, даты проживания, данные гостя, статус).

## 6. Взаимодействие с внешними сервисами

| Сервис | Назначение | Направление |
|--------|-----------|-------------|
| Публичный API Bnovo (`public-api.reservationsteps.ru`) | Получение данных: номера, цены, доступность, тарифы, услуги, промокоды | Чтение (GET) |
| Модуль бронирования Bnovo (`reservationsteps.ru`) | Создание бронирований | Запись (POST) |
| Платёжный шлюз Bnovo (`payment.bnovo.ru`) | Онлайн-оплата бронирования | Redirect пользователя |

## 7. Результат работ

По итогам разработки Заказчику передаются:

1. **Виджет бронирования** — работающее React-приложение с пользовательским интерфейсом.
2. **Серверная часть** — Node.js приложение с API-прокси и административной панелью.
3. **Код встраивания** — фрагмент HTML-кода для размещения виджета на сайте Заказчика.
4. **Инструкция по размещению** виджета на сайте.
5. **Инструкция по работе** с административной панелью.

## 8. Требования к хостингу

Для работы виджета необходим сервер (предоставляется Заказчиком или арендуется Исполнителем по согласованию) со следующими характеристиками:

- Операционная система: Linux.
- Node.js версии 18 или выше.
- Доменное имя с SSL-сертификатом (HTTPS).
- Доступ к интернету для обращения к API Bnovo.
