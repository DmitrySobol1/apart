# Documentation Language Convention

All project documentation (.tasks drafts, plans, subtasks, etc.) MUST follow this bilingual structure:
1. **English section (primary)** — used by the development agent for planning, implementation, and code generation. This is the authoritative section.
2. **Russian translation section (at the bottom)** — provided for the project owner's convenience. Clearly separated with a heading and a NOTE indicating it is for reference only.
3. The development agent MUST NOT read or use the Russian section for any development, planning, or implementation decisions.
4. When creating or updating documentation, always maintain both sections in sync.
